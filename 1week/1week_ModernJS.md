# 04장 변수 (Variable)

## 변수란?

- 하나의 값을 저장하기 위해 확보한 메모리 공간 또는 그 공간을 식별하기 위해 붙인 이름

## 변수가 필요한 이유

```javascript
var result = 10 + 20; // 결과값 30을 재사용 가능
```

- 값의 재사용
- 값의 의미 파악 용이
- 코드 가독성 향상

## 메모리

- 메모리 셀: 1바이트(8비트) 단위, 고유한 메모리 주소 보유
- 메모리 주소는 CPU와 OS가 자동으로 관리

## 변수와 메모리

```javascript
var result = 10 + 20;
// 1. 10 + 20 평가 → 30 생성
// 2. 메모리 공간에 30 저장
// 3. 변수 result가 메모리 주소를 가리킴
// 4. result로 메모리의 값 30에 접근
```

## 여러 값 저장

```javascript
// 객체
var user = { id: 1, name: "Lee" };

// 배열
var users = [
  { id: 1, name: "Lee" },
  { id: 2, name: "Kim" },
];
```

## 변수명 작성

- 저장된 값의 의미를 명확하게 전달하는 이름 필수
- 코드 가독성과 유지보수성 고려

## 식별자

- 식별자는 어떤 값을 구별해서 식별할 수 있는 고유한 이름
- 변수, 함수, 클래스 등의 이름은 모두 식별자
- 변수 이름으로 메모리 공간에 저장된 값에 접근

## 변수 선언

- `var`, `let`, `const` 키워드 사용
- ES6 이전: `var` 키워드만 사용 가능
- ES6 이후: `let`, `const` 키워드 도입

### var vs let/const

- var: 함수 레벨 스코프
- let, const: 블록 레벨 스코프

## 변수 선언 과정

```javascript
var score; // 1. 변수 선언
score = 80; // 2. 값의 할당
```

1. **선언 단계**: 변수명을 등록하여 자바스크립트 엔진에 변수 존재 인지
2. **초기화 단계**: 값을 저장할 메모리 공간 확보 및 undefined 할당

### undefined

- 자바스크립트에서 제공하는 원시 타입의 값
- 변수 선언 후 값을 할당하지 않았을 때의 초기값

## 변수 선언의 실행 시점과 변수 호이스팅

- 변수 선언은 소스코드가 순차적으로 실행되기 전에 먼저 실행됨
- 이를 **변수 호이스팅**이라 함
- var 키워드로 선언한 변수는 모든 선언문이 제일 위로 옮겨진 것처럼 동작

```javascript
console.log(score); // undefined (ReferenceError 발생 안함)
var score = 80;
```

## 값의 재할당

- var 키워드로 선언한 변수는 값을 재할당 가능
- const 키워드는 단 한 번만 할당 가능

## const 키워드

- 상수(constant) 선언에 사용
- 선언과 동시에 초기값 필수
- 재할당 불가

## 식별자 네이밍 규칙

### 네이밍 컨벤션

- **카멜 케이스(camelCase)**: firstName, lastName
- **스네이크 케이스(snake_case)**: first_name, last_name
- **파스칼 케이스(PascalCase)**: FirstName
- **헝가리언 케이스**: strFirstName (type + identifier)

# 05장 표현식과 문

## 값(Value)

- 표현식이 평가되어 생성된 결과

## 리터럴(Literal)

- 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기법
- 값을 생성하기 위해 미리 약속한 표기법
- 리터럴의 종류로는 (정수 리터럴, 부동소수점 리터럴, 2진수 리터럴, 8진수 리터럴, 16진수 리터럴, 문자열 리터럴, 불리언 리터럴, null 리터럴, undefined 리터럴, 객체 리터럴, 배열 리터럴, 함수 리터럴, 정규 표현식 리터럴 등)이 있다.

## 표현식(Expression)

- 값으로 평가될 수 있는 문
- 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조
- 표현식은 값으로 평가되며, 이미 표현식과 표현식이 평가된 값은 동등함
- ex) `1 + 2 = 3`, `1 + 2`는 3과 같고 3은 1 + 2와 같음

## 문(Statement)

- 프로그래밍 언어에서 실행 가능한 최소 단위
- 명령문이라고도 함
- 문은 0개 이상의 표현식으로 구성
- 완료되지 않은 문은 다음과 같이 구분되고 해석됨
  - 변수 선언문: 변수 선언 후 변수를 참조하는 경우
  - 할당문: 값 할당
  - 조건문: 지정한 조건에 따라 실행 코드 블록 실행
  - 반복문: 반복 코드 실행

### 문의 종류

- **선언문**: var, let, const, function, class
- **할당문**: x = 5;
- **조건문**: if (x > 1) { console.log(x); }
- **반복문**: for (var i = 0; i < 2; i++) { console.log(i); }

## 세미콜론과 세미콜론 자동 삽입 기능

- 세미콜론(;)은 문의 종류를 나타냄
- 자바스크립트 엔진은 세미콜론으로 문이 종료된 위치를 파악하고 순차적으로 실행
- 세미콜론 자동 삽입(ASI, automatic semicolon insertion) 기능으로 세미콜론을 생략 가능

## 표현식인 문과 표현식이 아닌 문

- **표현식인 문:** 값으로 평가될 수 있는 문 (ex. var x = 100;)
- **표현식이 아닌 문:** 값으로 평가될 수 없는 문 (ex. var x;)
- 변수 선언은 표현식이 아님
- 다시 말해 같은 값으로 평가된 값은 다른 표현식의 일부가 될 수 없음
- 표현식인 문은 표현식이 아닌 문이 될 수 없음

# 06장 데이터 타입

## 데이터 타입

- 자바스크립트의 모든 값은 데이터 타입을 가짐
- ES6는 7가지 데이터 타입 제공
  - **원시 타입**: 숫자, 문자열, 불린, undefined, null, 심볼
  - **객체 타입**: 객체, 배열, 함수 등

## 숫자 타입

- C, Java 등과 달리 하나의 숫자 타입만 존재
- 정수, 실수, 2진수, 8진수, 16진수 모두 숫자로 처리
- 64비트 부동소수점 형식을 사용
- 정수만을 위한 타입이 없고 모든 수를 실수로 처리
- Infinity, -Infinity, NaN 이 세 가지의 특별한 값도 표현 가능
- JavaScript는 대소문자를 구별하기 때문에 Not A Number을 반드시 'NaN'으로 표기

## 문자열 타입

- 텍스트 데이터를 나타내는 데 사용
- 0개 이상의 16비트 유니코드 문자(UTF-16)의 집합으로 전 세계 대부분의 문자 표현 가능
- 작은따옴표(''), 큰따옴표(""), 또는 벡틱(``)으로 감싸서 표현. 작은따옴표가 일반적.
  - 문자열을 따옴표로 감싸는 이유는 키워드나 식별자 같은 토큰과 구분하기 위함

## 템플릿 리터럴

- ES6부터 도입된 새로운 문자열 표기법
- 멀티라인 문자열, 표현식 삽입, 태그드 템플릿 등 제공
- 백틱(`)으로 표현

### 멀티라인 문자열

- 일반 문자열 내에서는 줄바꿈(개행)이 허용 X
- 백슬래시(\)로 시작하는 이스케이프 시퀀스로 줄바꿈 등의 공백 표현
- but, 템플릿 리터럴 내에서는 이스케이프 시퀀스를 사용하지 않고도 줄바꿈 및 공백이 그대로 적용

### 표현식 삽입

- ES5에서는 '+'로 문자열끼리 연결
- 템플릿 리터럴 내에서는 **표현식 삽입**, ${}으로 표현식을 감싸서 문자열을 표현.

## 불리언 타입

- 불리언 타입의 값은 논리적 참, 거짓을 나타내는 true와 false 밖에 없음

## undefined 타입

- undefined 타입의 값은 undefined가 유일
- var 키워드로 선언한 변수는 암묵적으로 undefined로 초기화
- 개발자가 의도적으로 할당할 값으로 사용하지 않음

## null 타입

- null 타입의 값은 null이 유일
- 프로그래밍 언어에서 null은 변수가 이전에 참조하던 값을 더 이상 참조하지 않음을 의도적으로 명시

### null과 undefined의 차이

- **null**: 변수에 값이 없다는 것을 의도적으로 명시
- **undefined**: 변수를 참조했을 때 선언은 되었지만 값을 할당하지 않음

## 심벌 타입

- ES6에서 추가된 7번째 데이터 타입
- 변경 불가능한 원시 타입의 값
- Symbol() 함수로 생성

## 객체 타입

- 자바스크립트의 데이터 타입은 크게 원시 타입과 객체 타입으로 분류
- 함수, 배열, 정규 표현식 등도 객체에 포함

## 데이터 타입의 필요성

### 데이터 타입이 필요한 이유

- 값을 저장할 때 확보해야 하는 메모리 공간의 크기를 결정하기 위해
- 값을 참조할 때 한 번에 읽어 들어야 할 메모리 공간의 크기를 결정하기 위해
- 메모리에서 읽어 들인 2진수를 어떻게 해석할지 결정하기 위해

## 동적 타이핑

### 동적 타입 언어와 정적 타입 언어

- **정적 타입 언어**: C, Java 등 - 변수 선언 시 타입을 명시
- **동적 타입 언어**: JavaScript - 변수의 타입이 런타임에 결정

### 동적 타입 언어와 변수

- 동적 타입 언어는 유연하지만 신뢰성이 떨어질 수 있음
- 변수는 타입을 갖지 않지만 값은 타입을 가짐
- typeof 연산자로 변수의 데이터 타입 확인 가능
